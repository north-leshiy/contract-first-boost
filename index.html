Code First: что хорошо

  • Мы автоматизировали часть процесса
  • Контакт будем максимально синхронизирован с реализацией
  • Позволяет работать параллельно (или с небольшой задержкой)
  • Ранее появление тестового сервера позволяет уменьшить время на отладку

Прокачиваем автоматизацию Frontend'a

  • Верстка компонентов
  • Написание и поддержка типов + валидация
  • Написание и поддержка слоя взаимодействия с backend (client)
  • Написание остальной логики

Прокачиваем автоматизацию Frontend'a

Выделяем куски работы Frontender'a

  • Верстка компонентов
  • Написание и поддержка типов + валидация [авто!]
  • Написание и поддержка слоя взаимодействия с backend (client) [авто!]
  • Написание остальной логики

Прокачиваем автоматизацию Frontend'a

  • Генерируем из контракта типы данных
  • Генерируем из контракта клиент
  • Серьезно уменьшаем количество ошибок на уровне
    • Cтат анализа IDE
    • Проверок консистентности CI/CD
  • На выходе серьезно сокращаем объем работы и ускоряем работу фронтов

Выводы

  1. Оба подходы имеют право на жизнь, их можно серьезно прокачать
  2. Но contract first имеет больше гибкости
  3. В больших компаниях contract first более актуален
  4. Не ленитесь внедря автоматизации на основе контракта, они серьезно уменьшают количество ошибок и ускоряют работу
